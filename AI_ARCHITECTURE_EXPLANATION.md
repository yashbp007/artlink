# Artist Platform AI Architecture & Technology Explanation

This document is a comprehensive guide to understanding the AI architecture of the Artist Platform. It explains **what** technologies are used, **why** they were chosen, and **how** they work together across the frontend (React), backend (Spring Boot), and AI service (Python).

---

## 1. High-Level Architecture

The system uses a **Microservice Architecture** consisting of three main parts:

1. **Frontend (React)**: The user interface. It only communicates with the Spring Boot backend.
2. **Main Backend (Spring Boot / Java)**: The brain of the application. It handles user authentication, business logic, MySQL database operations (users, posts, relations), and MongoDB operations (AI metadata).
3. **AI Service (Python / FastAPI)**: A specialized, stateless microservice dedicated purely to running Machine Learning models. It only talks to Spring Boot and its own Vector Database (ChromaDB).

### Why this split?
- **Python** is the industry standard for AI and Machine Learning. Running models in Java is very difficult and inefficient.
- **Java/Spring Boot** is excellent for robust business logic, transactions, and security.
- By keeping the Python service **stateless** (meaning it doesn't modify MySQL or MongoDB), we avoid nasty synchronization bugs. Python just calculates scores and vectors, returns them, and Spring Boot decides what to do with them.

---

## 2. The AI Models Used

We use open-source, pre-trained models from the Hugging Face ecosystem. This means no external API costs (like OpenAI), and data privacy is maintained since everything runs locally.

### A. Text Moderation: `Detoxify`
- **What it does**: Analyzes text (captions, comments, bios) and scores it across 6 categories: toxicity, severe toxicity, obscene, threat, insult, identity attack.
- **Why we use it**: It's a fast, highly accurate model built specifically for moderating online communities. It helps us automatically flag or block hate speech and spam.

### B. Image Understanding: `CLIP` (OpenAI's CLIP model via Hugging Face)
- **What it does**: CLIP (Contrastive Language-Image Pretraining) understands how text and images relate to each other.
- **How we use it for Moderation (Zero-Shot Classification)**: We give CLIP an image and ask it, "Does this image contain explicit content, violence, hate symbols, weapons, or graphic material?" It gives us a probability score for each.
- **How we use it for Tagging**: If an image is safe, we ask CLIP, "Which of these genres (hip hop, jazz, digital art, etc.) or moods best matches this image?" It automatically tags the image without the user having to type anything.

### C. Text Embeddings: `SentenceTransformers` (`all-MiniLM-L6-v2`)
- **What it does**: Converts text (like a post's caption + its tags + its genres) into a **Vector** (a list of 384 numbers).
- **Why we use it**: Computers don't understand text, but they understand numbers. By converting posts into vectors, we can mathematically calculate how "similar" two posts are by measuring the distance between their vectors. The `MiniLM` model is chosen because it is extremely fast and lightweight while maintaining high quality.

---

## 3. The Vector Database: `ChromaDB`

- **What it is**: A specialized database designed purely for storing and querying Vectors (the lists of numbers generated by SentenceTransformers).
- **How we use it**: When a post is published, its vector is saved in ChromaDB along with metadata (likes, comments, age). When we need to generate a feed for a user, we convert the user's interests into a vector, ask ChromaDB to find the "closest" post vectors, and it returns highly relevant posts instantly.
- **Why not MySQL?**: Standard relational databases (MySQL) cannot efficiently search for "semantic similarity" across millions of vectors. ChromaDB is built specifically for this.

## 4. The 3 Core AI Services

The Python application exposes three main workflows via REST APIs (built with FastAPI):

### Service 1: Content Moderation
1. User uploads a post in React.
2. Spring Boot receives it and pauses.
3. Spring Boot sends the text and image to Python.
4. Python runs `Detoxify` (text) and `CLIP` (image).
5. Python calculates if the content is `SAFE`, `REVIEW` (needs admin check), or `BLOCK`.
6. Spring Boot saves this result to the database and either blocks the post or proceeds.

### Service 3: Content Tagging & Embedding (Runs only if SAFE)
1. Spring Boot sends the safe post back to Python.
2. Python uses `CLIP` to generate tags, genres, and moods.
3. Python takes the caption + tags + genres and uses `SentenceTransformers` to create a Vector.
4. Python saves the Vector into `ChromaDB`.
5. Spring Boot saves the tags/genres into MySQL and MongoDB, and marks the post as `PUBLISHED`.

### Service 2: Feed Recommendation
1. User opens the app; React asks Spring Boot for the Feed.
2. Spring Boot looks up the user's favorite genres, recent searches, and liked tags from MySQL/MongoDB.
3. Spring Boot sends this user context to Python.
4. Python queries `ChromaDB` to find posts that match this context mathematically.
5. Python runs a custom **Hybrid Scoring Formula** (combining genre match, semantic similarity, freshness, and popularity) to rank the posts.
6. Python returns a ranked list of Post IDs to Spring Boot.
7. Spring Boot fetches the actual Post data for those IDs from MySQL and sends the finalized feed back to React.

---

## 5. The Spring Boot Databases

While Python handles the math, Spring Boot handles the persistence:

- **MySQL**: The primary source of truth. It stores Users, Posts, Genres, Follows, and User Interactions (Likes, Views, Saves). It uses relational tables ensuring data integrity.
- **MongoDB**: The unstructured data store. AI metadata (like the list of specific tags, the exact moderation scores, and search history) is stored here. Storing this in MySQL would require complex JSON columns or excessive joining. MongoDB allows us to quickly dump and retrieve complex AI payloads.

## Summary for the Backend Engineer

- The Python code is just a smart calculator. It takes inputs, runs ML models, and returns scores/tags.
- You (in Spring Boot) are the orchestrator. You call Python at the right times, save its answers, and make the final business decisions.
- Follow the `INTEGRATION_GUIDE.md` in the `spring-boot-integration` folder to see the exact Java code needed to link Spring Boot to Python.
